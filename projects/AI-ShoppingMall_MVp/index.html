AI ShoppingMall MVP

럭셔리 의복·장신구 중심 추천형 커머스 플랫폼

ncott.shop은 Next.js 15, React 19, Tailwind CSS 4를 기반으로 구현한 AI 쇼핑몰 MVP입니다. 인증, 주문, 결제, 배송, 운영관리까지 전 과정을 하나의 흐름으로 묶고, 추천·결제·물류 연동을 실제 서비스 수준으로 검증하는 데 초점을 맞췄습니다.

Google/Kakao SNS 로그인, iron-session + JWT 혼합 인증, TossPayments 승인/웹훅, PostgreSQL + Prisma + pgvector 추천, S3 Presigned URL + CDN, 배송 Provider Adapter 구조를 적용해 MVP 이후 확장 가능한 기본 뼈대를 완성했습니다.

프로젝트 개요

분야: AI 추천 기반 쇼핑몰 MVP
역할: 풀스택 설계 및 구현
핵심 목표: 추천 + 결제 + 배송 추적 통합
운영 관점: 관리자 대시보드 중심 확장 설계

기술 스택

Next.js 15 React 19 Tailwind CSS 4 TypeScript Prisma pgvector PostgreSQL TossPayments iron-session JWT AWS S3 + CDN Docker

핵심 기능

  • 로그인: Google·Kakao·일반 로그인 동시 지원
  • 결제: 승인 API + 웹훅으로 상태 동기화
  • 추천: 벡터 유사도 + 카테고리 fallback
  • 배송: 라이더 이벤트 기반 실시간 추적
  • 운영: 매출/주문/재고 통합 모니터링

아키텍처 & 처리 흐름

1

프런트엔드 렌더링 (Next.js 15)

SSR/CSR 혼합 전략으로 초기 진입 속도와 인터랙션 품질을 동시에 확보하고, 모바일·태블릿·PC 반응형을 통합 제공.

2

인증 계층 (iron-session + JWT)

세션과 토큰을 혼합해 로그인 상태 유지 안정성과 API 확장성을 함께 확보. SNS 로그인까지 동일 권한 모델로 처리.

3

주문·결제 트랜잭션

주문 생성, 재고 반영, 결제 승인/취소 흐름을 트랜잭션으로 묶어 정합성을 유지하고 장애 시 복구 가능성을 높임.

4

추천 엔진 (pgvector)

유사도 검색을 우선 적용하고, 결과 공백 발생 시 카테고리 fallback으로 보완해 추천 비노출 구간을 최소화.

5

배송 연동 (Provider Adapter)

외부/내부/Mock 공급자를 어댑터 계층으로 분리해 구현 비용을 낮추고, 라이더 위치 로그 기반 추적 신뢰성을 강화.

6

운영 최적화 (캐시 + 워밍)

API 캐시 제어와 워밍 전략을 적용해 최신성과 초기 응답속도를 균형 있게 유지하고, 운영 대시보드에서 KPI를 실시간 확인.

핵심 성과

🔐

혼합 인증 구조

SNS 로그인과 일반 로그인을 하나의 인증 체계로 통합해 진입 장벽을 낮추고 운영 일관성을 확보.

💳

결제 안정성 강화

TossPayments 승인 API와 웹훅을 연동하고 주문·결제 트랜잭션을 적용해 실패/중복 처리 리스크를 최소화.

🧠

추천 공백 최소화

pgvector 기반 유사도 추천과 카테고리 fallback을 함께 적용해 추천 부재 상황을 실무적으로 보완.

🎞️

미디어 표현력 확대

이미지뿐 아니라 동영상 상품까지 지원해 프리미엄 카탈로그 표현력과 전환 가능성을 강화.

🚚

실시간 배송 추적

라이더 위치 로그와 배송 이벤트를 기록해 배송 가시성을 높이고 고객 신뢰도를 개선.

🧩

연동 유연성

Provider Adapter 패턴으로 외부/내부/Mock 환경을 빠르게 전환해 개발과 운영의 민첩성을 확보.

📊

운영 대시보드

매출, 주문 상태, 재고 부족 상품을 통합 모니터링해 MVP 이후 운영 확장까지 고려한 구조를 구현.

⚙️

배포/비용 유연성

Docker 개발 환경 분리와 Supabase DB 전환 고려로 운영 안정성과 비용 유연성을 동시에 준비.
